## Задача 3: Клас Rational (Алгоритми и аритметика)

**Напишете клас Rational за аритметични пресмятания с рационални числа:**
* **Данни:** Целочислени `numerator` (числител) и `denominator` (знаменател).
* **Конструктори и методи:**
    * Пълен набор конструктори и SET/GET методи.
    * **Опростяване:** Конструкторът и SET методите трябва да опростяват числото чрез Най-голям общ делител (НОД) (напр. 2/4 става 1/2).
    * **Нормализация:** Знаменателят винаги трябва да бъде положително число (напр. 1/-2 става -1/2).
    * Конструктор по подразбиране: задава 1/1.
    * Метод `toString()` във формат `числител/знаменател`.

* **Операции (public методи):**
    * а) Събиране
    * б) Изваждане
    * в) Умножение
    * г) Делене
    * *Забележка: Всички резултати трябва да са в опростен вид.*

## Задача 4: Клас SavingsAccount (Static members)

**Напишете клас SavingsAccount:**
* **Член-данни:**
    * `mAnnualInterestRate` (статична променлива за годишната лихва).
    * `mSavingsBalance` (баланс по сметката).
    * `mNumber` (програмно генериран уникален номер на сметката).
* **Функционалност:**
    * Трите вида конструктори.
    * SET и GET методи.

    * Статичен метод `modifyInterestRate(double newRate)`.
    * Метод `isGreater(SavingsAccount acc)`: сравнява текущия баланс с този на друг обект.
* **Тестване:**
    * Създайте два обекта, сравнете ги.
    * Създайте копие на обект и го сравнете с оригинала.
